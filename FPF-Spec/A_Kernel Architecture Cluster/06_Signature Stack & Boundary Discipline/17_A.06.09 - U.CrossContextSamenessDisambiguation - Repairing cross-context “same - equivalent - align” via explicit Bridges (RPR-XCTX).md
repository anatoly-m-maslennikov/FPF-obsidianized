## [[17_A.06.09 - U.CrossContextSamenessDisambiguation - Repairing cross-context ‚Äúsame - equivalent - align‚ÄĚ via explicit Bridges (RPR-XCTX)|A.6.9]]:2 - Problem

When an umbrella predicate is used as if it were a single relation, readers (and downstream editors) silently choose defaults:

* **Symmetry hallucination:** ‚Äúequivalent‚ÄĚ is read as symmetric even when the intended relation is ‚äĎ/‚äí (directional).
* **Scope creep:** ‚Äúalign/map‚ÄĚ is read as substitution‚ÄĎeligible, leaking into Role Assignment & Enactment or Concept‚ÄĎSet row scopes.
* **Loss erasure:** ‚Äúsame‚ÄĚ implies lossless transfer even when units, granularity, preconditions, or stance differ.
* **License confusion:** ‚Äúexplain X using Y‚ÄĚ is mistaken for ‚ÄúY can stand in for X‚ÄĚ.
* **Implicit inversion:** later prose uses the inverse direction without an explicit redeclaration, breaking the ‚Äúno silent inversion‚ÄĚ rule.

The result is not merely imprecise wording: it changes what inferences are considered safe, and it pollutes Concept‚ÄĎSet row scopes via unnoticed weakest‚ÄĎlink violations.

It also breaks **temporal coherence**: if the underlying canons (glossaries, schemas, code lists, ontologies) evolve, an un‚ÄĎpinned ‚Äúequivalent‚ÄĚ claim silently becomes a claim about *two different editions at once*.
