entry cue, README scenario, ToC query cue, or local pattern `Problem frame` when
first-pattern choice is high-risk, often misclassified, repeatedly failed,
retrieval-facing, or too dense for compact guidance.
Its [[E.11 - First-Practical Entry and Pattern-Use Discoverability Discipline|E.11]] use is limited to disambiguation, wrong-pattern rejection,
entry-load reclassification, and admissible entry stop.

An `ExpandedEntryDisambiguationCase` is a bounded entry case. It is not
`U.Work`, not a workflow, not a route script, and not an execution trace.

[[I.2 - Expanded Entry Disambiguation Cases|I.2]] carries expanded disambiguation only when README scenarios, ToC query cues,
[[E.11 - First-Practical Entry and Pattern-Use Discoverability Discipline|E.11]] entry-distribution cues, or local `Problem frame` recognition are
